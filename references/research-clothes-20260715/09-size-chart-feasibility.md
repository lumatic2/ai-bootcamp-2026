## Go/No-Go 판정
- 확보 성공: **11개 / 시도 17개** (무신사스탠다드, 커버낫, 스파오, 마크곤잘레스, 예일, 스투시, 라코스테†, 널디†, 디스이즈네버댓, 아디다스, 노스페이스) — †라코스테는 라벨 매핑 실패, 널디(NDY)는 소매 수치 이상으로 재검증 필요 → **즉시 사용 가능 9개**
- 판정: **GO** — 실패 브랜드도 대부분 "실측 사이즈 탭이 없는 상품을 골랐기 때문"이라, 실측 제공 SKU로 재시도하면 회수 가능 (내일 오전 30분이면 15개+ 여력)
- 크롤링 방식: 무신사는 SSR에 사이즈표가 없어 정적 fetch 실패 → **headless 브라우저 기반(scrapling stealthy_fetch)만 성공**. "기준표 사이즈"(국제 공용표, 무의미)와 "실측 사이즈" 탭을 반드시 구분해야 함.

## 수집 실패와 회수 경로
| 브랜드 | 실패 이유 | 회수 경로 |
|---|---|---|
| 지오다노·칼하트·탑텐·뉴발란스 | 고른 상품에 "실측" 탭 없음("기준표"만) | 실측 제공 SKU로 재시도 — 사람이 URL만 골라주면 즉시 |
| 나이키 | 잘못된 상품 ID → 리다이렉트 | URL 재확인 |
| 폴로 랄프 로렌 | 30초 타임아웃(봇 방어 의심) | 재시도 |
| 후아유·유니클로·빈폴 | 미착수/검색 실패 | 공식몰 직접 |

## 구현 결정 입력 (정규화 스펙)
1. 표준 스키마: 총장/어깨/가슴**단면**/소매 (cm). 둘레 표기 브랜드는 ÷2 환산 + `chestMeasuredAs: "circumference"` 태그
2. 어깨 결측 허용 (래글런) — 매칭 로직은 존재하는 치수만으로 가중 비교
3. 사이즈 라벨 3체계 혼재 (알파벳 / 090~110 / 자체코드) — 원표기 보존 + 정렬용 순번 부여
4. 무신사 스크래핑은 headless 브라우저 필수, "실측 사이즈" 탭만 채택
