## 지식 3층 분류
- **L1 (검증된 관행)**: 옷 도메인에서 브랜드마다 사이즈 표기(44/55/66, S/M/L)가 실측과 어긋난다는 것은 이미 널리 알려진 문제이며, "실측 사이즈 표기"가 그 대응책으로 정착되어 있다 [E3] https://www.khan.co.kr/article/201507200927031 · Size Korea가 한국인 인체표준 데이터를 제공하고 'Size Korea' 인증 마크가 존재한다는 것도 기존 인프라다 [E2-근접/정부기관] https://sizekorea.kr/page/data/1_1 . 이 두 사실을 "우리가 처음 발견한 문제"인 것처럼 제시하면 안 된다 — 재발명 금지.
- **L2 (신흥 트렌드)**: 딥러닝 기반 사이즈 추천(임베딩+협업 필터링), Transformer 기반 사이즈 예측(PreSizE, 2021), 그래프 기반 아웃핏 호환성 추천(GNN), 리뷰 텍스트에서 fit feedback을 NLP로 추출하는 연구가 2018~2024년 사이 활발하다 [E2] 다수 (아래 근거 참조). 과열 주의: 이 연구들은 모두 "자사 카탈로그+구매 이력"이라는 대량 프로프라이어터리 데이터를 전제로 하며, 하루 만에 우리가 재현 가능한 조건이 아니다.
- **L3 (제1원리 발견)**: **"사이즈는 속성이 아니라 관계다"** — Amazon(Sembium et al., WWW 2018)과 후속 연구들이 공통으로 채택한 프레임은 "고객 사이즈"와 "상품 사이즈"를 절대 수치가 아니라 각자의 **잠재변수(latent size)**로 놓고 그 차이를 모델링한다. 이는 "고객 체형 실측값 → 브랜드 사이즈표 매칭"이라는 통념(옷 업계의 사이즈 가이드 관행)이 원천적으로 틀렸다는 것을 학술적으로 증명한 사례다: 실측 표기가 정확해도 브랜드마다 재단선·핏감이 달라 "같은 실측=같은 핏"이 성립하지 않는다. 즉 업계 관행(실측 사이즈 표기 확대)은 필요조건이지만 충분조건이 아니며, 문제의 본질은 신체 대 의류의 "상대적 관계"이지 "절대 치수의 정확성"이 아니다. [E2] https://dl.acm.org/doi/10.1145/3178876.3186149 (WWW 2018) — 이 통찰은 여전히 인용되는 기반 연구(stale이지만 유효, 후속 논문들이 2019~2024까지 이 프레임을 계승함, 예: PreSizE https://dl.acm.org/doi/10.1145/3404835.3462844, Tailor https://arxiv.org/html/2401.01978).

## 후보 1: 브랜드 간 사이즈 상대성 계산기 (Amazon 프레임의 이식)
- **유추 출처 또는 논문 근거**: Sembium et al., "Bayesian Models for Product Size Recommendations", WWW 2018 — 고객/상품을 잠재 사이즈 변수로 모델링, fit outcome(Small/Fit/Large)을 예측
- **왜 옷에선 아직 안 됐나**: 방법은 이미 있음(2018년부터). 안 된 이유는 "우리 손에 있는 데이터 부재" — Amazon은 구매 후 fit feedback(반품 사유 "사이즈가 작았어요" 등)을 대량 보유하지만, 팀이 접근 가능한 공개 한국 쇼핑몰 fit feedback 데이터셋은 확인 못함
- **관찰된 사실**: 같은 '55' 표기 셔츠도 브랜드 간 가슴둘레가 86cm vs 108cm로 22cm 차이 (경향신문 조사)
- **근거**:
  - [E2] Bayesian Models for Product Size Recommendations — https://doi.org/10.1145/3178876.3186149 — 2018 (stale이지만 후속 연구의 기반, 여전히 인용됨)
  - [E2] A Deep Learning System for Predicting Size and Fit in Fashion E-Commerce (Zalando) — arXiv:1907.09844 — 2019-07-23 (stale, 기반 연구)
  - [E3] "같은 표기 사이즈도 최대 20㎝ 차이" 경향신문 — https://www.khan.co.kr/article/201507200927031 — 2015 (stale, 여전히 인용되는 문제 사례)
- **MVP 힌트**: 사용자가 평소 입던 브랜드+사이즈를 입력하면 다른 브랜드에서의 상응 사이즈를 추정해 보여주는 웹 폼 형태가 상상됨 (사실만, 평가 아님)
- **모르는 것**: 한국 쇼핑몰에서 공개 fit feedback/리뷰 텍스트를 하루 안에 스크래핑 가능한 규모로 확보할 수 있는지 미확인. LLM으로 "이 브랜드 사이즈 리뷰"를 합성 생성해 대체할 경우 신뢰도 문제 있음.

## 후보 2: 반품 사유 텍스트에서 "핏 불만" 자동 추출 (리뷰 마이닝)
- **유추 출처 또는 논문 근거**: Baier, "Analyzing Customer Feedback for Product Fit Prediction", arXiv:1908.10896, 2019-08-28
- **왜 옷에선 아직 안 됐나**: 방법(NLP 전이학습, ULMFit)은 있음. 저자 스스로 "추출된 정보를 실제 사이즈 추천 시스템에 통합하는 것은 향후 과제로 남긴다"고 명시 — **미해결 지점이 정확히 여기(통합 단계)**. 즉 방법 부재라기보다 "적용·통합 부재"
- **관찰된 사실**: fit feedback은 "드물고 대개 비정형 텍스트로만 존재한다"(원문 인용)
- **근거**:
  - [E2] Analyzing Customer Feedback for Product Fit Prediction — arXiv:1908.10896 — 2019-08-28 (stale, 기반 연구로 여전히 인용)
- **MVP 힌트**: 리뷰 텍스트를 붙여넣으면 "작아요/커요/맞아요" 감정을 LLM으로 분류해 보여주는 데모 형태가 상상됨
- **모르는 것**: 한국어 리뷰 데이터로 이 분류가 영어권 논문 수준 정확도를 낼지, 한국 쇼핑몰 리뷰를 하루 안에 수집 가능한 규모로 얻을 수 있는지 미확인.

## 후보 3: 의류 성분(소재) 기반 필터링 서비스 부재 ("옷의 화해")
- **유추 출처 또는 논문 근거**: 화장품 산업의 화해(성분 크라우드소싱+등급화) 유추 이식
- **왜 옷에선 아직 안 됐나**: **표준 부재 + 아무도 안 함**. 화장품은 "전성분표기제"라는 법적 의무 규제가 성분 데이터의 존재 자체를 강제한다. 의류는 소재 혼용율 표기 의무는 있지만(예: KC 라벨), 개인이 "이 폴리에스터 소재가 나한테 땀띠를 유발한다"는 반응성 데이터를 쌓을 표준 채널이 없음. 규제 조건이 없어 화장품과 달리 이식이 구조적으로 어려움 — 명시적으로 이 조건 부재를 인지해야 함
- **관찰된 사실**: 화해는 화장품 성분·리뷰·랭킹 정보를 제공하는 국내 최대 플랫폼이나, 의류 버전 부재 이유에 대한 공식 자료는 검색으로 확인 못함
- **근거**:
  - [E5] 검색 결과, 공식 설명 없음 — https://www.hwahae.com/en
- **MVP 힌트**: 소재 혼용율을 입력하면 "민감 피부에 좋은/나쁜 소재"를 알려주는 간단 필터 폼이 상상됨 (근거 데이터가 약해 신뢰도 낮음)
- **모르는 것**: 소재-피부반응 상관관계에 대한 검증된 데이터셋이 존재하는지 전혀 확인 못함. 이 후보는 데이터 기반이 가장 약함 — L1에서 지적한 "조건 없음"이 강하게 적용됨.

## 후보 4: 아웃핏(코디) 호환성 추천 — 개인화와 호환성의 이중 최적화 난제
- **유추 출처 또는 논문 근거**: Han et al., "Learning Fashion Compatibility with Bidirectional LSTMs", 2017; Li et al., "Hierarchical Fashion Graph Network for Personalized Outfit Recommendation", RecSys 2020
- **왜 옷에선 아직 안 됐나**: 방법 부재보다는 **"동시 최적화의 난제"** — Li et al.(2020)는 선행 연구들이 "user-outfit 관계 또는 outfit-item 관계 중 하나만 다뤄 최적이 아닌 표현으로 이어진다"고 명시. 즉 스타일 호환성(아이템 간)과 개인 취향(유저별)을 동시에 잘 푸는 방법이 아직 미완성
- **관찰된 사실**: Alibaba iFashion의 POG(2019)는 실서비스 규모에서 생성+추천을 결합한 Transformer 구조를 씀
- **근거**:
  - [E2] Learning Fashion Compatibility with Bidirectional LSTMs — https://doi.org/10.1145/3123266.3123394 — 2017 (stale이지만 기반 연구로 여전히 인용, 313회 피인용)
  - [E2] POG: Personalized Outfit Generation — https://doi.org/10.1145/3292500.3330652 — 2019 (stale, 기반 연구)
  - [E2] Hierarchical Fashion Graph Network — https://doi.org/10.1145/3397271.3401080 — 2020 (stale, 기반 연구)
- **MVP 힌트**: 옷장 사진 여러 장을 업로드하면 LLM이 어울리는 조합을 골라주는 데모가 상상됨
- **모르는 것**: 이 연구들은 모두 대규모 자체 데이터셋(Alibaba, Polyvore) 기반 — 하루 만에 팀이 재현 가능한 소규모 데이터로 이 방법들이 그대로 작동할지 미확인.

## 후보 5: 가상 피팅(Virtual Try-On)의 원단 물리 시뮬레이션 격차
- **유추 출처 또는 논문 근거**: Han et al., "VITON", CVPR 2018; Yu et al., "VTNFP", ICCV 2019
- **왜 옷에선 아직 안 됐나**: **방법 부재** — 2D 이미지 기반 접근은 3D 정보 없이 옷을 "왜곡(warp)"시켜 합성하는데, 원문에서 VITON은 "초기 합성 결과가 흐릿하다(blurry)"는 한계를 명시하고, VTNFP는 "신체와 의류 디테일을 동시에 보존하며 사실적인 이미지를 생성하는 것"이 핵심 난제라고 명시. 원단의 물리적 질감·주름 재현은 여전히 미해결
- **관찰된 사실**: 여러 논문에서 공통으로 "흐릿한 합성 영역", "섬세한 원단 질감·패턴 보존의 어려움"을 한계로 명시
- **근거**:
  - [E2] VITON: An Image-Based Virtual Try-on Network — https://doi.org/10.1109/cvpr.2018.00787 — 2018 (stale, 기반 연구)
  - [E2] VTNFP — https://doi.org/10.1109/iccv.2019.01061 — 2019 (stale, 기반 연구)
- **MVP 힌트**: 하루 만에 원단 물리 시뮬레이션 수준까지 구현은 어렵고, 기존 API(예: 이미지 생성 모델)로 단순 합성 데모만 상상 가능
- **모르는 것**: 최신(2024~2026) 상용 가상피팅 API의 실제 정확도 수준을 확인 안 함 — 이 분야는 학계보다 산업이 빠르게 앞서갔을 가능성 있음, 별도 확인 필요.
