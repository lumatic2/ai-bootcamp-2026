# FORME Fashion Commerce — Design Specification

## 0. 목적과 참고 범위

이 명세는 무신사 상품 탐색 경험에서 **정보 밀도, 내비게이션 계층, 상품 정보 위계, 그리드, 필터·정렬, 반응형 전환 방식**만 추상화해 독자적인 패션 쇼핑몰에 적용한 것이다. 무신사의 로고, 상표, 고유 문구, 실제 상품 이미지, 전용 아이콘, 색 조합, 레이아웃을 픽셀 단위로 복제하지 않는다.

직접 URL은 자동화 환경에서 글로벌 지역 선택 화면으로 전환되고 브라우저 접근도 차단되어 실제 화면의 픽셀 측정은 불가능했다. 따라서 아래 수치는 공개된 UI·UX 설명과 일반적인 고밀도 패션 커머스 패턴을 토대로 **새롭게 정의한 구현 명세**다.

참고 근거:
- 상단 상시 검색, 카테고리 내비게이션 단순화, 상품·스냅·콘텐츠로 검색 결과 구조를 정리한 공개 설명
- PC 해상도에서 탐색 도구와 상품 정보량을 확대하고, 상품 상세와 장바구니를 분할 레이아웃으로 구성했다는 공개 설명
- 상품 리스트에서 목적별 필터와 추천·판매·가격 등 다양한 정렬을 제공한다는 공개 설명

---

## 1. 디자인 원칙

1. **고밀도이되 스캔 가능해야 한다.** 상품 카드 간 장식은 최소화하고 브랜드, 상품명, 가격, 할인율, 평점만 즉시 비교되도록 한다.
2. **검색과 카테고리 이동은 항상 짧은 경로를 제공해야 한다.** 데스크톱에서는 검색창과 카테고리 바를 상시 노출하고, 모바일에서는 헤더와 가로 스크롤 카테고리로 축약한다.
3. **상품 이미지가 1차, 가격이 2차, 부가 정보가 3차다.** 브랜드명과 상품명은 가격보다 작거나 같은 크기지만 위쪽에 배치해 읽는 흐름을 유지한다.
4. **필터는 데스크톱에서 상시 노출하고, 태블릿·모바일에서는 드로어로 전환한다.**
5. **브레이크포인트마다 단순 축소가 아니라 정보 구조를 재배치한다.**

## 2. 컬러 시스템

| 토큰 | 값 | 용도 |
|---|---:|---|
| `--background` | `#FCFCFA` | 전체 배경 |
| `--surface` | `#FFFFFF` | 카드, 헤더, 필터 패널 |
| `--foreground` | `#171717` | 기본 텍스트 |
| `--muted-foreground` | `#737373` | 보조 정보, 리뷰 수 |
| `--border` | `#E7E5E4` | 구분선, 입력창 테두리 |
| `--subtle` | `#F3F2EF` | 칩, 비활성 배경 |
| `--primary` | `#12382C` | 주요 버튼, 활성 상태 |
| `--primary-foreground` | `#F8FFF9` | 주요 버튼 텍스트 |
| `--accent` | `#D9FF70` | 세일·신상품 강조 배경 |
| `--danger` | `#D93C32` | 할인율, 오류 |
| `--success` | `#287A4D` | 배송 가능, 재고 상태 |

색상 사용 규칙:
- 한 화면에서 강한 강조색은 `primary`, `accent`, `danger` 세 종류까지만 사용한다.
- 상품 카드의 80% 이상은 무채색으로 유지한다.
- 할인율만 `danger`, 최종 가격은 `foreground`, 정가는 `muted-foreground`로 구분한다.

## 6. 헤더와 카테고리 내비게이션

### 6.1 데스크톱

총 높이 144px:
1. Utility bar: 32px
2. Main header: 64px
3. Category navigation: 48px

Main header 구성:
- 로고 텍스트: 112×28px 내외
- 검색창: 폭 420~520px, 높이 42px, 좌우 패딩 14px
- 아이콘 버튼: 40×40px, 아이콘 20px
- 요소 간 간격: 12~20px

Category navigation:
- 수평 메뉴, 높이 48px
- 메뉴 간 간격 24px
- 활성 메뉴 하단 2px 선
- 스크롤 시 상단 고정, `z-index: 40`

### 6.2 태블릿

총 높이 108px:
- Main header 64px
- Category navigation 44px
- Utility bar 제거
- 검색창은 280~360px로 축소

### 6.3 모바일

총 높이 96px:
- Main header 56px
- Category navigation 40px
- 로고 88×24px 내외
- 검색은 아이콘 버튼으로 축약하거나 검색 결과 페이지에서 전체 폭 입력창으로 확장
- 카테고리는 가로 스크롤, 좌우 fade 없이 스냅하지 않는다

## 7. 상품 카드

### 7.1 비율과 크기

- 카드 폭: 그리드에 따라 유동
- 이미지 비율: `4:5`
- 이미지 최소 폭: 모바일 160px, 데스크톱 180px
- 이미지 배경: 연한 중성색
- 카드 사이 세로 간격: 28~36px
- 이미지와 텍스트 간격: 10px

### 7.2 정보 위계

1. 이미지
2. 브랜드명 — 12px/600, 1줄
3. 상품명 — 13~14px/400, 최대 2줄
4. 가격 행 — 할인율 14px/700, 판매가 15px/700
5. 정가 — 12px/400, 취소선
6. 메타 정보 — 평점, 리뷰 수, 배송 또는 배지, 11~12px

### 7.3 상호작용

- 이미지 호버 시 `scale(1.015)`, 220ms ease-out
- 카드 전체를 링크로 사용하되 찜 버튼은 독립 버튼으로 둔다
- 찜 버튼: 우측 상단 36×36px, 모바일 32×32px
- 상품 배지: 좌측 하단, 높이 22px, 패딩 6px 8px, 10~11px/600

## 8. 상품 그리드

### 상품 목록 페이지

- Desktop/Wide: `236px filter + 32px gap + minmax(0, 1fr)`
- Wide 상품 열: 5열, 열 간격 16px
- Desktop 상품 열: 4열, 열 간격 16px
- Tablet: 3열, 열 간격 14px
- Mobile: 2열, 열 간격 10px
- 행 간격: 데스크톱 40px, 모바일 28px

### 홈 페이지

- Desktop/Wide: 5열
- Tablet: 3열
- Mobile: 2열
- 추천 섹션은 동일 카드 컴포넌트를 재사용한다.

## 10. 페이지별 레이아웃

### 10.1 메인 페이지

1. Hero: 데스크톱 620px 높이, 모바일 520px
2. 빠른 카테고리 칩: 데스크톱 72px, 모바일 가로 스크롤
3. 추천 상품: 5/3/2열
4. 에디토리얼 배너: 2열 → 모바일 1열
5. 신상품: 5/3/2열

### 10.2 상품 목록 페이지

- 상단 여백 32px
- Breadcrumb 12px
- H1 + 상품 수
- 카테고리 칩
- 필터/정렬 툴바
- 데스크톱 좌측 필터 + 우측 상품 그리드

### 10.3 상품 상세 페이지

Desktop:
- 12열 그리드
- 이미지 영역 7열, 구매 패널 5열
- 이미지 2열 갤러리
- 구매 패널 `position: sticky; top: 170px`
- CTA 버튼 56px 높이

Tablet:
- 이미지 1열 또는 2열, 구매 패널 하단 이동

Mobile:
- 단일 이미지 캐러셀 형태
- 구매 정보는 본문 흐름에 배치
- 하단 고정 CTA 64px

### 10.4 검색 결과 페이지

- 검색어를 H1보다 먼저 인지할 수 있도록 28px 굵기로 표시
- 연관 검색어 칩
- 상품 목록 구조 재사용

- 좌측 상품 목록 `minmax(0, 1fr)`

## 12. 접근성 및 상태

- 모든 아이콘 버튼에 `aria-label`을 제공한다.
- 포커스 링은 2px `primary`, 바깥 여백 2px를 사용한다.
- 텍스트 대비는 WCAG AA 이상을 목표로 한다.
- 이미지에는 상품명 기반 대체 텍스트를 사용한다.
- 빈 검색 결과, 장바구니 비어 있음, 로딩 상태를 각각 제공한다.
- 터치 대상은 최소 40×40px, 핵심 CTA는 48px 이상으로 유지한다.
