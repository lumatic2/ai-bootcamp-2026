# 01-size-translator-accuracy — 브랜드 관계 신호만으로 사이즈를 맞힐 수 있는가

> `experiments/01-size-translator-accuracy/README.md` — 실험은 *가설·방법·결과·통찰* 4섹션.
> Problem A(사이즈 도박) 확정(2026-07-15 팀 미팅) 후의 코어 실험. 리서치 근거: `references/research-clothes-20260715/`.

## 1. 가설 (Hypothesis)

"사용자의 실측 치수 없이, **잘 맞는 브랜드+사이즈라는 관계 신호**(+ 실측표 매핑 + 리뷰 핏 신호)만으로 처음 보는 브랜드의 추천 사이즈를 **적중률 60% 이상**(우연 수준 ~33%: S/M/L 3지선다 가정)으로 예측할 수 있다."

반증: 블라인드 검증 적중률 40% 미만이면 엔진 가설 기각 → S8(브랜드 핏 성향 카드)로 전환하고 기각 과정을 발표.

하위 가설 E0 (선행, 오늘): "브랜드 상의 실측표를 오후 3시간 내 10개 브랜드 이상 확보할 수 있다." — 실패 시 리뷰 마이닝(S2)을 코어로 승격.

## 2. 방법 (Method)

### 셋업 (2026-07-16 현행화 — 측정 대상은 당일 배포본)
- 데이터: 시드 120브랜드 + **상품 단위 실측 차트 245개**(`src/data/product-charts.json` — 팀 시트 수기 + 크롤러 옵션열 라벨 복구, 교차검증 로그는 changeset #2step-rebuild)
- 엔진: 클라이언트 실행 `translateToRows()` — 앵커(잘 맞는 옷) 평균 벡터를 각 상품 실측표에 가중 매칭 (가슴3·어깨2·총장1·소매0.5)
- UI: 2스텝 (Step1 브랜드→상품→사이즈 앵커 + AI 사이즈표 검색 + 실측 직접입력 옵션 / Step2 상품 그리드+검색). 리뷰 마이닝은 2026-07-15 범위 제외로 미사용
- AI 요소: 시드 밖 브랜드의 사이즈표 웹검색·파싱(`/api/parse-chart`, gpt-5.4-mini) — 커스텀 앵커 경로

### 시나리오 (mock 먼저, real 다음)
- V0: 팀원 4명이 자기 옷(브랜드+사이즈 아는 것)으로 교차 예측 — 정답을 아는 상태의 파이프라인 점검
- V1: 부트캠프 참가자·지인 15~20명 블라인드 검증 — "잘 맞는 브랜드·사이즈 하나 알려주면 다른 브랜드 사이즈를 맞혀볼게" (모집=측정)
  - ⚠ 실제 수행분(2026-07-16, 팀원 주관 Google Form): 블라인드 적중률 검증이 아니라 **문제 빈도·판단 기준·사용 의향 설문**으로 실행됨. 적중률 신호는 이 설문에서 나오지 않으며, 적중률 측정은 V0 원장 + GA4 `feedback_fit`이 담당. raw: `data/v1-survey-2026-07-16.csv`

### 측정 metric (KPI 정의: `docs/presentation-2026-07-15.md`)
- 핏 적중률(★ North Star, 목표 ≥60%) / 첫 예측 완료율 ≥50% / 첫 결과까지 ≤60초 / 결정 채택률 ≥40% / 당일 재조회율 ≥25% / 예측불가율 ≤30% (가드레일)
- 계측: GA4 이벤트 (`start_input`→`view_result`/`view_grid`→`adopt_size`(카드 클릭=`outbound_click` 동시 발화)/`repeat_query`, 적중률은 `feedback_fit {hit|miss}`) — ⚠ GA는 07-16 오전까지 프로덕션 미주입 상태였음(changeset #prod-env-fix), 실측정은 07-16 오후분부터 유효
- V0 수기 기록: `V0-log.md` (예측 vs 실착 원장 — GA와 교차 대조)

### V1 설문 상세 (n=16, 2026-07-16 13:08~14:32, Google Form)
- **사이즈 판단 기준**: 체형 비슷한 사람 후기 9명(56%) > 과거 구매 사이즈 4명(25%) > 실측 치수 비교 3명(19%)
- **사이즈 불편 빈도**: 10번 중 1~2번 14명(88%) / 3번 이상 1명(6%) / 없다 1명(6%) → **94%가 사이즈 실패 경험**
- **사용 의향**: 네 15명(94%) / 아니요 1명(6%)
- **데모 열람 후 반응**: "한 번 사용해보고 **정확도만 높다면** 계속 사용" 15명(94%) / "딱히 필요 없을 것 같다" 1명(6%)
- 자유 의견 4건: 긍정 3(ㄷㄷ·멋져요·최곱니다) / 부정 1(최악이노 — 유일한 무불편·무의향 응답자와 동일인)
- **한계**: n=16 소표본, 부트캠프 참가자 편중, 보기 선택지가 거칠어(1~2번/3번+/없다) 유도성 있음, 적중률 블라인드 검증 아님

## 4. 통찰 (Insights)

### 무엇을 알아냈나
- (V1 설문분, 2026-07-16) **문제는 실재**: 94%가 사이즈 실패를 경험하고, 94%가 사용 의향 — 단 전원이 "정확도만 높다면"을 조건으로 걸었다. **적중률이 North Star라는 KPI 설계가 사용자 언어로 재확인됨.**
- (V1 설문분) 사람들의 현재 해법은 후기 의존(56%)이 1위, 실측 비교는 19%뿐 — 너비가 자동화한 "실측 비교"는 소수 고관여 사용자의 기술이었고, 이를 후기 의존층에게 대신해주는 것이 포지셔닝. 리뷰 마이닝(S2)을 차기 후보로 유지할 근거이기도 함.
- (남은 것) 적중률 자체는 이 설문으로 검증되지 않음 — V0 원장 + GA `feedback_fit` 필요.
