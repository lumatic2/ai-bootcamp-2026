# mvp-build-day — 너비 MVP 팀 빌드데이 런북 (2026-07-16)

> 4명이 한 노트북(운전자 1 + 항법사 3)으로 thin slice를 완성하고, 당일 배포·측정까지 닫는 하루 절차.
> 전제: step-1 스캐폴딩 완료(커밋 `4b8e796`) — 3스텝 플로우·번역 엔진·시드 9브랜드·GA4 이벤트가 이미 동작한다. 오늘 빌드데이의 일은 "만들기 시작"이 아니라 **데이터 보강 → 검증 → 배포 → 측정**이다.

## 1. 입력

- 레포 `main` 최신 (`git pull`), `pnpm install` 완료 상태
- `.env.local`: `OPENAI_API_KEY`(완료) + `NEXT_PUBLIC_GA_MEASUREMENT_ID=G-42Z5R1WD7Z`(완료)
- 팀 구글시트 실측 데이터 (컬럼 계약: `브랜드 | 상품명 | URL | 사이즈라벨 | 총장 | 어깨 | 가슴단면 | 소매` — cm, 가슴 둘레 표기 시 ÷2 후 기입)
- Vercel 배포 URL (step-2 — 미완이면 §2-0에서 처리)
- 검증 참가자: 팀원 4명(V0) + 지인·참가자 15~20명 리스트(V1)

## 2. 절차 (시간은 안내선, 순서가 계약)

**§2-0. [09:30~10:00] 배포 확인** *(결정론적)*
- step-2가 전날 막혔으면: Vercel 토큰으로 프로젝트 생성 → env 2개 주입 → 배포 → `bootcamp.askewly.com` CNAME. 검증: 프로덕션 URL 200 + `/api/translate` 실호출 1건.

**§2-1. [10:00~11:00] V0 — 팀원 교차검증 (가장 먼저, 코드 만지기 전)** *(판단 필요)*
- 각 팀원이 "잘 맞는 브랜드+사이즈"를 입력하고, **자기가 실제 소유한 다른 브랜드 옷**으로 예측 vs 실착 결과를 기록한다 (1인 2~3조합, 목표 표본 ≥8).
- 기록: `experiments/01-size-translator-accuracy/README.md` 결과 표 V0 행 + 화면의 [맞았어요/틀렸어요] 버튼(= GA `feedback_fit`).
- **분기**: 적중률 ≥60% → §2-2로. 40~60% → 신뢰도 임계값(`src/lib/translate.ts` 주석 표시부)을 보정하고 재검증. **<40% → 중단, 반증 발동** — 원인 분석을 실험 기록에 남기고 발표 전략을 "Learn 스토리"로 전환 (플레이북 3-day §Learn 절차).

**§2-2. [11:00~13:00] 시드 보강** *(결정론적)*
- 팀 시트 → `src/data/size-charts.json` 병합 (라벨 원표기 유지, 가슴은 단면 기준, 래글런 등 비교 불가 치수는 null).
- 무신사에서 "실측 사이즈" 탭 있는 상품만 채택 ("기준표 사이즈" 금지 — E0 함정).
- 목표: 9 → 15개+ 브랜드. 검증: `pnpm check` PASS + 신규 브랜드 1개당 번역 1회 스모크.

**§2-3. [13:00~15:00] 마감 품질** *(판단 필요)*
- 모바일 실기기(팀원 폰 4대) 렌더·터치 확인, 문구 다듬기, 오답 시 사과 문구 톤 확인.
- 리뷰 마이닝 실전 테스트: 무신사 실상품 리뷰 복붙 3건 — 인용 왜곡 없는지 눈으로 대조 (LLM 환각 가드).
- 커밋 단위: 응집 변경마다 1커밋, `pnpm check` 통과 후에만.

**§2-4. [15:00~17:00] V1 — 블라인드 배포·측정** *(결정론적 수집 + 판단 금지)*
- 배포 URL QR 생성 → 부트캠프 참가자 + 지인 카톡 발송 (모집 스크립트: "잘 맞는 옷 브랜드·사이즈 하나만 알려주면, 다른 브랜드 사이즈를 맞혀볼게").
- 목표 표본: 예측 15건+, feedback_fit 10건+.
- 관측만 한다 — 중간에 UX 고치지 않는다 (측정 오염 방지). 발견 결함은 메모로 큐잉.

**§2-5. [17:00~18:00] 데이터 마감 + Learn** *(판단 필요)*
- GA4 실시간 보고서 + 서버 로그에서 KPI 6종 값 추출 → `experiments/01` 결과 섹션 기입.
- 통찰 섹션 작성 (비어 있으면 실험 미종료 — repo 규약). PASS/FAIL 판정과 다음 가설 1개.
- 발표 evidence 경로 정리 → `docs/presentation-2026-07-15.md` §3 실측값 채움.

## 3. 체크리스트 (측정 가능 항목만)

- [ ] V0 표본 ≥8, 적중률 수치 기록됨 (분기 판정 완료)
- [ ] 시드 브랜드 ≥15, 신규 브랜드별 번역 스모크 1회씩
- [ ] `pnpm check` 최종 PASS + 프로덕션 URL 200
- [ ] V1 예측 ≥15건, feedback_fit ≥10건 (GA4 실시간 보고서 스크린샷 박제)
- [ ] `experiments/01` 4섹션 모두 채움 (통찰 포함)
- [ ] KPI 6종 실측값이 `docs/presentation-2026-07-15.md` §3 표에 기입됨
- [ ] 오답 사례 최소 1건을 발표 자료에 그대로 노출 (정직성 = 루브릭 대응)
