## 문제 (Problem A — 확정)
온라인에서 처음 보는 브랜드 옷을 살 때, 사이즈를 못 믿어 도박하듯 주문하고 반품한다.
- E1: 패션플랫폼 4사 피해구제 278건(2022)→592건(2024), 사유 1위 청약철회 48.4% (한국소비자원)
- 같은 'M'도 브랜드 간 가슴단면 최대 5cm+ 차이 (시드 데이터 실측 확인)

## 사용자 (페르소나)
온라인(무신사류)에서 처음 보는 브랜드 옷을 사는 20대 중후반 남성. 자기 실측 치수를 모르고, 사이즈 실패로 반품하거나 여러 사이즈를 동시 주문해본 경험이 있음.
반(反)페르소나: 오프라인 시착 위주 구매자, 같은 브랜드만 재구매하는 사람.

## 핵심 가설 (experiments/01)
실측 치수 입력 없이 "잘 맞는 브랜드+사이즈" 관계 신호만으로 추천 사이즈 적중률 ≥60% (우연 ~33%). 40% 미만이면 기각·피벗.

## MVP 계약
사용자가 ① 잘 맞는 브랜드+사이즈를 고르고 ② 사려는 브랜드를 고르면(+선택: 리뷰 붙여넣기) → **추천 사이즈 + 신뢰도(상/중/하) + 근거 2종(실측 비교표, 리뷰 핏 신호)**를 30초 안에 돌려준다.

- 지원 범위: 남성 반팔 티셔츠 1개 카테고리, 시드 9개 브랜드 (의도적 축소 — 적중률 우선)
- AI 코어: 리뷰 텍스트 핏 신호 추출(LLM, `/api/mine-reviews`) — LLM 없이는 성립 불가
- 엔진: 브랜드 실측표 가중 거리 매핑(가슴3·어깨2·총장1·소매0.5), `/api/translate`
- 사이즈 라벨: 화면에는 대상 브랜드 원표기, 내부 비교는 cm 실측값만
- 예측 불가 UX: 지원 브랜드만 선택지로 노출 (실패 경로 최소화)

## 측정 (KPI — docs/presentation-2026-07-15.md §3이 정본)
GA4 이벤트 6종: `start_input` → `view_result`(elapsed_ms) → `adopt_size` / `feedback_fit`(correct) / `repeat_query` / `predict_unavailable`. North Star = 핏 적중률(feedback_fit correct 비율 + V0/V1 블라인드 검증).

## MVP 제외 사항
- 하의·아우터·신발 카테고리 (2단계)
- 회원가입·인증·DB (이벤트 계측으로 충분)
- 크롤링 자동화 파이프라인 (수동 큐레이션 시드로 충분)
- 도박 비용 공유 카드 (2026-07-15 제외 확정)
- 코디·구매 추천 (원안의 잔여 욕구 — 관계 데이터 축적 후 2단계 로드맵)

## 성공 지표
- V1 블라인드(15~20명) 핏 적중률 ≥60%, 첫 결과까지 ≤60초, 첫 예측 완료율 ≥50%
- 발표에서 문제→가설→제품→데이터→학습이 레포 evidence 경로로 역추적된다
