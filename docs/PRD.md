# PRD — Bootcamp Experiment Workspace

## 목표
팀이 제품 아이디어와 무관하게 문제 정의부터 측정·학습·발표까지 같은 절차와 증거 구조를 사용하게 한다.

## 사용자
- 부트캠프 팀원: 문제와 솔루션을 함께 결정하고 3일 안에 제품을 구현한다.
- 발표 평가자: 아이디어보다 결정·플래닝·검증 과정을 빠르게 추적한다.

## 핵심 기능
1. 문제·가설·실험·학습의 필수 입력과 단계별 통과 조건을 제공한다.
2. Next.js, Tailwind, shadcn 기반의 아이디어 비종속 웹 스타터를 제공한다.
3. DESIGN.md와 검증 명령으로 UI 결정을 코드와 연결한다.
4. 발표에 필요한 결정·측정 evidence 위치를 미리 정의한다.

## MVP 제외 사항
- 실제 제품 기능: 팀의 문제·사용자 결정 전 구현하면 선택을 왜곡한다.
- 백엔드·인증·DB·AI API: 최소 실험이 요구할 때만 추가한다.
- 특정 브랜드 시각 언어: 팀이 제품 맥락과 함께 선택한다.
- 배포: 로컬 통합 검증 후 발표 방식이 정해질 때 결정한다.

## 성공 지표
- 새 팀원이 플레이북만 보고 15분 안에 첫 문제 후보와 증거를 기록할 수 있다.
- 핵심 가설마다 metric, 수집 방법, 판정 기준이 모두 존재한다.
- starter의 design lint, code lint, production build가 재현 가능하게 통과한다.
- 발표 자료의 주요 주장마다 레포 evidence 경로가 연결된다.
