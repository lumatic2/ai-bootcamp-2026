# ADR 0002 — 임시 DESIGN.md 계약

- 상태: Accepted
- 날짜: 2026-07-14

## 맥락
UI 일관성과 자동 검증은 미리 필요하지만, 제품·사용자·브랜드는 팀 논의 전이다.

## 결정
Google `design.md` 구조에 맞는 중립적 임시 `DESIGN.md`를 사용한다. `VoltAgent/awesome-design-md`는 레퍼런스 카탈로그로만 사용하고 브랜드 파일을 그대로 복제하지 않는다.

## 근거
- 기계 판독 토큰과 사람용 디자인 이유를 한 문서에서 관리할 수 있다.
- lint로 깨진 참조와 대비 문제를 구현 전에 찾을 수 있다.
- 임시 토큰임을 명시하면 팀의 디자인 선택을 보존하면서도 starter를 검증할 수 있다.

## 결과
- 제품 방향이 결정되면 현재 디자인과 후보 레퍼런스를 비교하고 새 ADR과 함께 토큰을 교체한다.
- `DESIGN.md` 규격은 alpha이므로 CLI 버전 변경 시 diff와 lint 결과를 재확인한다.
