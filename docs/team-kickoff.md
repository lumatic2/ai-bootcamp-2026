# 팀 킥오프 — 첫 45분

## 0–10분: 문제 후보를 사실로 적기
각 팀원은 아래 카드를 최대 2개 작성한다.

```text
문제 후보:
누가/언제 겪는가:
직접 본 사실 또는 원문:
현재 대안:
모르는 점:
증거 위치:
```

## 10–25분: 후보 비교
각 후보를 문제 강도, 빈도 신호, 사용자 접근성, 3일 내 검증 가능성, 팀 역량으로 1~5점 표시한다. 점수는 토론 입력이며 자동 의사결정기가 아니다.

| 후보 | 강도 | 빈도 | 사용자 접근 | 3일 검증 | 팀 역량 | 가장 큰 불확실성 |
|------|------|------|-------------|----------|---------|------------------|
| | | | | | | |

## 25–35분: 첫 가설과 측정
```text
우리는 [사용자]가 [상황]에서 [문제]를 겪는다고 믿는다.
[최소 제품/행동]을 제공했을 때 [metric]이 [기준]이면 가설을 지지한다.
반박/판단불가 조건:
raw data 저장 위치:
```

## 35–45분: 최소 Build와 역할
- 반드시 있어야 측정 가능한 surface:
- 만들지 않을 것:
- 사용자 접근 담당:
- 구현 담당:
- 데이터 기록 담당:
- 시간별 재집결 시점:
